# pw-core Documentation (v1.0.0)

This file contains the detailed documentation and usage guide for `pw-core`.

---

# Page Registry

Not every page requires a custom Page Object. Many pages only contain URLs, testIds, and selectors. For those pages, use `createPageRegistry`.

```ts
import { createPageRegistry } from "pw-core/page";

export const test = createPageRegistry({
  loginPage: {
    url: "/login",

    testIds: {
      username: "username-input",
      password: "password-input",
      submit: "login-button",
    },
  },
});
```

## Automatic Fixtures

Pages registered in `createPageRegistry` immediately become fixtures in your test runner:

```ts
test("login", async ({ loginPage }) => {
  await loginPage.fill("username", "admin");
  await loginPage.fill("password", "password");
  await loginPage.click("submit");
});
```

No Page Object class or custom fixture setup is required!

# Extending Registry Pages

As pages become more complex, you can extend them with custom classes:

```ts
class LoginPage extends test.pages.loginPage {
  async login() {
    await this.fill("username", "admin");
    await this.fill("password", "password");
    await this.click("submit");
  }
}
```

Update the fixture:

```ts
export const extendedTest = test.extend({
  loginPage: LoginPage,
});
```

Usage:

```ts
await loginPage.login();
```

Start simple and extend only when needed.

# Readable Assertions

Traditional Playwright assertions require wrapping elements in `expect(locator)` blocks:
```ts
await expect(page.getByTestId("login-button")).toBeVisible();
```

With `pw-core`, assertions are chainable and directly accessible:
```ts
await login.verify("loginBtn");
```

Custom assertions remain available:
```ts
await login.verify("loginBtn").toBeEnabled();
await login.verify("errorMessage").toContainText("Invalid");
```

# Chained Locators

Complex nested locators stay highly readable regardless of depth.

Traditional:
```ts
await page
  .locator('[data-testid="modal"]')
  .locator('[data-testid="form"]')
  .locator('[data-testid="submit"]')
  .click();
```

pw-core:
```ts
await page.click("modal.form.submit");
```

# Reusable Components

With `pw-core`, components become first-class citizens. Define once and reuse across different pages:

```ts
export class TransactionsPage extends TypedPage<typeof config> {
  constructor(page: Page) {
    super(page, config);
    this.transactionsTable = new Table<Transaction>(tableLocator);
  }
}
```

# AI-Friendly Automation

`pw-core` standardizes and simplifies the interaction pattern to make it highly friendly for AI agents.

Instead of writing custom combinations of `page.locator()`, `page.getByRole()`, `page.getByTestId()`, all actions follow a standardized, predictable API:

```ts
page.click(...)
page.fill(...)
page.verify(...)
page.locator(...)
```

Benefits:
- Reduced context size & token usage
- Better and more predictable AI-generated tests
- Faster onboarding and easier code reviews

# TypedPage

`TypedPage` is the foundation of `pw-core`. It transforms page configurations into fully typed automation objects.

```ts
import { TypedPage, createPageConfig } from 'pw-core/page';

export const config = createPageConfig({
  url: "/login",
  testIds: {
    username: "username-input",
    submitBtn: "login-button",
  },
});

class LoginPage extends TypedPage<typeof config> {
  async login(user: string, pass: string) {
    await this.fill("username", user);
    await this.click("submitBtn");
  }
}
```

---

# When To Use TypedPage vs Page Registry

### Use TypedPage when:
- The page contains complex workflows
- The page contains reusable business logic
- Multiple tests interact with the page

### Use Page Registry when:
- Pages only contain locators
- No custom page methods are needed
- You want minimal boilerplate and automatic fixtures

# Traditional Playwright vs pw-core Comparison

## Traditional Approach

```ts
class LoginPage {
  constructor(private page: Page) {}

  username = this.page.getByTestId("username-input");
  password = this.page.getByTestId("password-input");
  loginButton = this.page.getByTestId("login-button");
  errorMessage = this.page.getByTestId("login-error");
}
```

Usage:

```ts
await loginPage.username.fill(user);
await loginPage.password.fill(pass);
await loginPage.loginButton.click();

await expect(loginPage.errorMessage).toBeVisible();
```

---

## pw-core Approach

```ts
const config = createPageConfig({
  url: "/login",

  testIds: {
    username: "username-input",
    password: "password-input",
    loginBtn: "login-button",
    loginError: "login-error",
  },
});

class LoginPage extends TypedPage<typeof config> {
  constructor(page: Page) {
    super(page, config);
  }
}
```

Usage:

```ts
await loginPage.fill("username", "admin");
await loginPage.fill("password", "password");
await loginPage.click("loginBtn");
await login.verify("loginError");
```
